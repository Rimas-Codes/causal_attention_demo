What is causal self-attention?

The mechanism behind every decoder transformer.

At every layer of a GPT model, each token needs to gather context from the tokens that came before it. Self-attention does this by computing three projections of each token — a Query, a Key, and a Value — then blending the past tokens' Values, weighted by how well each Key matches the current Query.

// the four-line transformer attention recipe
scores = (Q · K⊤) / √dk
scores = mask(scores, future → −∞)
weights = softmax(scores)
output = weights · V
"Causal" means each token can only look backward in time — never at future tokens. Enforced by a triangular mask. This is what makes GPT a decoder and what lets it be trained efficiently on every position at once.

04

The causal mask, visualised

Why future tokens cannot leak into the past.

The causal mask is a lower-triangular pattern. Before softmax, every cell above the diagonal is set to −∞, because e−∞ = 0 — so future tokens get exactly zero weight.

mask[i, j] = 0   // if j ≤ i (can attend)
mask[i, j] = −∞   // if j > i (blocked)

att[i, j] = softmax( scores[i, j] + mask[i, j] )
This guarantee is what lets GPT predict the next token without ever cheating by peeking ahead — and lets the loss be computed at every position in parallel during training.

Multi-head attention

One model · twelve specialists. See each head's pattern at a glance.

Instead of running attention once with the full n_embd dimension, GPT-2 splits it into n_head = 12 parallel heads, each operating on dk = n_embd / n_head = 64 dimensions. Each head can specialise.

Why divide by √dk ?

The scaling factor that keeps gradients alive.

Without scaling, dot products grow with dimension. A random unit vector in d dimensions has dot-product variance of d. At d=64, scores are roughly 8× larger than at d=1, pushing softmax into a near one-hot distribution — and the gradient vanishes. Dividing by √dk normalises variance back to ≈ 1.

Var(Q · K) ≈ dk   // large for large dims
std ≈ √dk
scale by 1/√dk   →   Var ≈ 1
